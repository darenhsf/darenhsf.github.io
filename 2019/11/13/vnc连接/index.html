VNC连接配置过程

开启VNC Server

在终端输入命令：

sudo raspi-config

raspi-config是Raspbian系统自带的配置工具，树莓派的很多必要功能都需要通过它进行设置。

一开始没想到终端也会出现这种类似图形界面

依次选择Interfacing Options —-> VNC上下左右控制选项，开启VNC服务

除了使用wifi管理查看 ip ，也可以输入 ifconfig(ssh连接其实已经知道ip了)

安装VNC服务

上一步只是开启了树莓派的vnc接口

  • 以下步骤最好先给树莓派换源。使得下载东西速度快点

sudo apt-get install tightvncserver

在完成安装后需要对VNC设置一个密码，在命令行输入： vncpasswd （先输入连接客户端密码两次，然后会询问是否设置一个只能查看桌面(view-only)密码

这么多密码我也不知道干嘛的，我把密码都设为树莓派的登录密码

完成后，在命令行输入： vncserver 启动VNC服务。

vnc连接

1.首先file新建一个连接

2.输入 ip 和表示连接的任意名字 -> ok

3.双击出现得电脑图标进行连接，双击之后会出现一个警告，点击continue

4.输入用户名pi和设置的密码连接

VNC连接出现问题

下面的两种问题出现一般都可以解决

  • 没有开启桌面自动登录

解决Cannot currently show the desktop

1 开启桌面自动登录

选择 Boot Options->Desktop->Desktop Autologin desktop确定就好

2.调整桌面默认分辨率（不必要，但是可以设置试试）

选择Advanced Options->Resolution->DMT Mode 确认即可

分辨率条

  • vnc服务没有正常开启

解决vnc服务没有正常开启

其实还是重复上面vnc连接步骤

先确认树莓派是否工作，ip不要搞错

sudo raspi-config

1.开启vnc服务

sudo apt-get install tightvncserver（没有下载的下载）

2.输入vncserver，开启服务

树莓派换源

1.修改软件更新源

第一行注释掉

加上这个中科大的软件源地址

deb http://mirrors.ustc.edu.cn/raspbian/raspbian/ stretch main contrib non-free rpi

注意 stretch的地方可能有的是buster,以原本的为主

所以其实修改http://...这个网址就行了

  • 中国科学技术大学
    Raspbian http://mirrors.ustc.edu.cn/raspbian/raspbian/

  • 阿里云
    Raspbian http://mirrors.aliyun.com/raspbian/raspbian/

  • 清华大学
    Raspbian http://mirrors.tuna.tsinghua.edu.cn/raspbian/raspbian/

将第一行注释掉,加上中科大的系统源地址

deb http://mirrors.ustc.edu.cn/archive.raspberrypi.org/debian/ stretch main ui
