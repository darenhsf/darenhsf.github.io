主题

数据包

构建数据包

  • 包(Packet)是TCP/IP协议通信传输中的数据单位，一般也称“数据包”。
  • TCP/IP协议是工作在OSI模型第三层(网络层)、第四层(传输层)上的，帧工作在第二层(数据链路层)

    scapy的数据包创建遵循的是TCP / IP协议

各个层的协议都有各自的创建函数，如IP（），TCP（），UDP（）等，不同层之间通过” /”来连接。

  • /运算符已用作两层之间的合成运算符。这样做时，下层可以根据上层重载一个或多个默认字段。（您仍然可以提供所需的值）。字符串可用作原始层。

Command Effect
raw(pkt) 整个数据包转换为二进制字符串 (assemble the packet)
hexdump(pkt) 十六进制转储格式 have a hexadecimal dump
ls(pkt) 有字段值列表 have the list of fields values
pkt.summary() 单行摘要 for a one-line summary
pkt.show() 包的进一步展示 for a developed view of the packet
pkt.show2() same as show but on the assembled packet (checksum is calculated, for instance)
pkt.sprintf() 用数据包的字段值填充格式字符串 fills a format string with fields values of the packet
pkt.decode_payload_as() changes the way the payload is decoded
pkt.psdump() draws a PostScript diagram with explained dissection
pkt.pdfdump() draws a PDF with explained dissection
pkt.command() 返回可以生成数据包的Scapy命令 return a Scapy command that can generate the packet

发送和接收数据包

发送

  • send（）函数将在第3层发送数据包。也就是说，它将为您处理路由和第2层。
  • sendp（）函数将在第2层起作用。由您选择正确的接口和正确的链路层协议。
  • 如果将return_packets = True作为参数传递，则send（）和sendp（）还将返回已发送的数据包列表。

发送&接收

返回了两个清单。第一个元素是一对列表（发送分组，应答），第二个元素是未应答包的列表。这两个元素是列表，

  • sr（）函数用于发送数据包和接收答案。该函数返回几个数据包和应答的数据包以及未应答的数据包。
  • 函数sr1（）是一种变体，仅返回一个应答发送的数据包（或数据包集）的数据包。数据包必须是第3层数据包（IP，ARP等）。
  • 函数srp（）对于第2层数据包（以太网，802.3等）执行相同的操作。如果没有响应，则在达到超时时将分配None值。
