发表于2019-11-13|更新于2021-10-15

D-H算法

迪菲-赫尔曼密钥交换（英语：Diffie–Hellman key exchange，缩写为D-H） 是一种安全协议。它可以让双方在完全没有对方任何预先信息的条件下通过不安全信道创建起一个密钥。

最简单，最早提出的这个协议使用一个质数p的整数模n乘法群以及其原根g。

算法过程原理

例如：

1.Alice与Bob协定使用质数 p=23 以及原根 g=5.

2.Alice选择一个秘密整数a=6, 计算A = g^a mod p并发送给Bob

  • A = 5^6 mod 23 = 8

3.Bob选择一个秘密整数b=15, 计算B = g^b mod p并发送给Alice。

  • B = 5^15 mod 23 = 19.

4.爱丽丝计算s = B ^a mod p

  • s=19^6 mod 23 = 2.

5.鲍伯计算s = A^ b mod p

  • s=8^15 mod 23 = 2.

$$
最终都得到了同样的值，因为在模p下 \g^{ab}=g^{ba}
$$

A，B，g，p是公开的

但外界要得到秘钥S必须还要知道a或b

也就是安全性转化 为 要求解A=g^a mod p的a或B=g^a mod p的b的难度

为了安全，必须使用非常大的a, b 以及 p

否则可以实验所有g^ab mod 23的可能取值(总共有最多22个这样的值, 就算a和b很大也无济于事)

如果 p 是一个至少 300 位的质数，并且a和b至少有100位长， 那么即使使用全人类所有的计算资源和当今最好的算法也不可能从g, p和g^a mod p 中计算出 a。这个问题就是著名的离散对数问题

安全性

在选择了合适的G和g时，这个协议被认为是窃听安全的。因为只能窃听到A,B

破解秘钥必须要求解a，b，即离散对数问题

G的阶应当是一个素数，或者它有一个足够大的素因子
