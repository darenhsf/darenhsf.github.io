路由器作用

发表于2019-11-05|更新于2021-10-15

路由器的核心作用

是尽最大可能将入站数据包按照数据包头部的目的IP地址，转发到其所在的网络

路由器的转发数据工作原理：

路由（routing）指的是中间设备（比如路由器）通过查找路由表对数据包进行转发的处理方式

  • 主要依靠路由表转发数据

路由表

  • 路由表（Routing Table）等价于转发数据库；

  • 路由表中的“路由条目”等价于转发数据的路径。

网络设备建立转发数据库的方式：

路由信息的三种来源

  • 对应上面

  • Proto:学习路由的协议；RIP / Direct / Static 等

路由优先级

  • 对应上面
  • Pre:优先级；100

路由器给每种路由信息渠道赋予的一个权重，

此权重值就叫做路由优先级

路由优先级值越小，代表这种类型的路由可信度越高

路由类型 路由表中标识 默认路由优先级
直连路由 Direct 0
OSPF路由 OSPF 10
静态路由 Static 60
RIP路由 RIP 100

路由度量值

  • 对应上面
  • Cost:开销 2

指去往目的网络的开销；
度量值越小越优先；
不同的路由协议参考计算度量值的因素不一样。

直连路由

当对应接口没有配置ip时，即使物理接口有连接，

IP路由表中不会出现任何与此接口相关的直连路由信息

此时对应接口状态为 Physical up Protocol down

​ 配置ip 激活对应接口

静态路由

  • 去往同一目的网络，优先级相同则可以实现负载，配置优先级不同则可以实现主备

动态路由

动态路由协议的两个类别：

  • 距离矢量型
  • 链路状态型路由协议

常用的动态路由协议

RIP

一种距离矢量型路由协议

OSPF

链路状态型路由协议

相比较于静态路由，

动态路由协议具有更强的可扩展性，具备更强的应变能力

因此动态路由协议更适合于大规模网络环境

扩展性强：

  • 就配置而言，随着网络规模的增加配置静态路由的难度也会随之加大
  • 而配置动态路由，则只需要宣告路由器的直连网段，相对简化得多。

应变能力：

  • 静态路由缺乏应变能力导致静态路由不适合在大型网络中单独使用；
  • 而动态路由则可以对网络产生异常而自动收敛和应变。

路由协议的分类

根据路由协议所使用的算法对路由协议进行分类：

  • 距离矢量型路由协议
  • 链路状态型路由协议

距离矢量型路由协议

通过DV路由协议学习到的远端网络，D和V就是路由器所了解的全部信息。

每个节点都定期地将它们的路由表传送给所有相邻节点，

路由表所包含的内容有：

  • 每条路径的目的地址——矢量；

  • 本节点到该目的地址的代价——距离；

**如 路由1 到 路由2 有3跳 **

链路状态型路由协议

路由器通过LS路由协议通告的链路状态信息中包括了全部关于这个网络的具体信息

从路由协议在发送路由更新信息时是否会携带掩码，将路由协议分为：

有类路由协议

无类路由协议
